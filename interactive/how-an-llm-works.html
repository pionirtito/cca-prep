The agent loop: model and harness, back and forth

The model itself can't read files or run code. A harness (Claude Code, the Agent SDK, your own script) does that. The model only ever emits text and structured tool calls; the harness executes them and feeds results back. Step through one real task.

Thinking mode: a scratchpad before the answer

"Thinking" is not a different model or a special architecture. It is the same next-token generation, but the model first writes out reasoning tokens — a scratchpad — and then writes the answer, which is conditioned on that scratchpad. More deliberation, same machine.

With newer Claude models this is adaptive: the model decides how much to think based on difficulty and an effort setting. Try it on a small multi-step problem.

prompt
A train leaves at 14:20 and arrives at 17:05. A faster train covers the same route 35 minutes quicker, leaving at the same time. When does it arrive?
Thinking tokens are generated and billed like any output, and they stay in the context to condition what comes next. On hard tasks they raise accuracy; on trivial ones they just add latency, which is why adaptive thinking exists.

Long context: tokenization, attention, and the KV cache

Everything the model can "see" lives in one context window — system prompt, tools, full history, files, its own thinking. There is no other memory. Every token in it is reprocessed on each turn and billed as input.

Drag the context size. Watch the cost and the attention work grow — attention compares every token with every other token, so its cost scales roughly with the square of length.

Are embeddings and the neural network the same thing?

It depends which embeddings — and conflating the two is the single most common source of confusion. There are two completely different objects that both turn text into vectors.

inside the model

Token embeddings

  • The model's first layer: a learned lookup table (vocabulary × ~5000 dims).
  • These vectors are weights — trained jointly with every other layer, end to end.
  • They live in the same vector space the layers operate in (the residual stream).
  • The output "unembedding" that produces logits is often the same matrix, reused.
a separate model

RAG embeddings

  • Produced by a different, dedicated model (Anthropic has none of its own; it recommends Voyage).
  • Used to index documents and run similarity search in a vector DB.
  • Completely independent of the LLM's weights — swappable, retrainable on its own.
  • Their output never enters the network as weights — only as retrieved text in the prompt.
The verdict: token embeddings and the network are the same object — embeddings are literally the network's input layer, learned together. "RAG embeddings" come from a separate, independent model and only ever feed the LLM as text inside the context window.
